# Contributing to BitByte Labs

Thanks for your interest in contributing! This repository hosts the BitByte Labs company site. The guidelines below keep changes consistent and easy to review.

## Getting started

1. **Fork and clone** the repository.
2. **Install dependencies** (if you add new tooling):
   ```bash
   npm install
   ```
3. **Run the site locally**:
   ```bash
   npm start
   ```
   Then visit `http://localhost:3000`.

## Making changes

- Keep edits small and focused. If you plan a larger refactor, open an issue first.
- Match existing code style and formatting in `public/`.
- Use semantic, descriptive naming and clear copy.

## Commit messages

Use concise, imperative messages such as:

```
Add dependabot configuration
Refine access section styling
```

## Pull requests

When opening a PR, include:

- A short summary of the change.
- Screenshots for visual updates.
- Testing notes (commands run or a note if none).

## Reporting issues

If you find a bug or a content issue, open an issue with:

- Steps to reproduce.
- Expected vs. actual behavior.
- Screenshots when relevant.

## Code of Conduct

Be respectful and collaborative. We follow the [Contributor Covenant](https://www.contributor-covenant.org/).
